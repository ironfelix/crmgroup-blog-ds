## HARD GATE — статья не пишется без Research Report

Вывести в чат **перед** написанием (иначе публикация запрещена):

```
## Research Report: [ТЕМА]
### 1. Целевой запрос
Запрос: «...» | Wordstat WS: ... | Интент: информационный
### 2. Конкурентный анализ (топ-5 в Яндексе) — читать полностью, не только H2
1. [URL] — ~N слов, угол: ..., чего нет: ...
Уникальный угол: ...
### 3. Ключевые факты (с источниками и уровнем)
- Факт (Источник, год) — T1/T2/[T3]/[ПРОВЕРИТЬ]
### 4. Структура H2
1. ... 2. ...
```

**Правила источников (0.3–0.7):**
- **T1** = оригинальный отчёт, название + год проверены → атрибутировать
- **T2** = крупное СМИ со ссылкой на отчёт → атрибутировать с оговоркой
- **T3** = знаю, но источник не верифицирован → `по оценкам рынка`, `как правило` — **без имени организации**
- **❌** = назвать источник, который не проверял — никогда
- Нет названия отчёта + года → понизить до T3
- Данные старше 3 лет → добавить `(данные [год])` в текст
- Глобальные данные ≠ российские → писать `в мировой практике`
- **«По данным наших клиентов» / «из нашего опыта»** — ТОЛЬКО если пользователь явно передал данные в задании
- Статья с `[ПРОВЕРИТЬ]` не публикуется

## Обязательные WP-блоки в каждой статье

**1. Лид-параграф — ВСЕГДА первый параграф:**
```html
<!-- wp:paragraph {"className":"is-lead"} -->
<p class="wp-block-paragraph is-lead">Текст лида...</p>
<!-- /wp:paragraph -->
```

**2. Тултипы `span.explain` — ≥5 на статью; в технических темах ≥8:**
```html
<span class="explain">ТЕРМИН<span class="explain-card"><strong>Полное название</strong>Определение 80–200 символов.</span></span>
```
- При ПЕРВОМ упоминании термина/аббревиатуры — обязателен
- При цитировании исследований (McKinsey, HubSpot, Bain...) — обязателен
- Не более 3 тултипов в одном абзаце; не вкладывать тултип в тултип

**3. FAQ в конце статьи — ≥4 вопросов:**
Только `<details>/<summary>` + JSON-LD Schema.org — **НЕ H3+параграф**.
Шаблон: раздел 26 в BLOGCRM_WRITING_RULES.md

**4. Нумерованный список:** только паттерн `num-list`, **никогда** стандартный `<ol>`.

**5. block-comment:** содержимое **не менять** — это живые цитаты реальных людей.

**6. ACF block-content-list:** ACF self-closing тег закрывается строго `} /-->` (не `-->>`)

## Критические правила текста

**Тире:** ≤4 в теле статьи (не считая заголовков). Ctrl+F «—» → посчитать.

**Запрещены навсегда:**
- «следует отметить», «важно понимать», «данный», «таким образом», «в данном контексте»
- «комплексный подход», «эффективный инструмент», «позволяет выстроить долгосрочные отношения»
- «С одной стороны... С другой стороны...»
- «является», «представляет собой», «осуществляет», «обеспечивает»
- Instagram, Facebook — **не упоминать** (запрещены в РФ)

**Факты без источника → не публиковать.** Источники T3 (`по оценкам рынка`) допустимы. «По нашей практике / данным клиентов» — только с явными данными от пользователя.

**Активный залог:** «Сегментация осуществляется» → «Мы сегментируем».

**Абзацы:** max 3 предложения. Предложения: цель 15–20 слов, максимум 30.

## Структурные анти-паттерны ИИ — проверять в каждой статье

**1. Первый H2 не должен дублировать H1** — тезис или угол, не пересказ.

**2. «Первый блок: / Второй блок:»** внутри абзацев — убирать. Нумерация только в `num-list`.

**3. Скобки вместо тире** в пояснениях: `❌ удержание (снижение оттока)` → `✅ удержание — снижение оттока`.

**4. Расшифровка аббревиатуры** — один раз. Если в следующих 3 абзацах уже есть → убрать из текущего места.

**5. Дубль контента между H2** — один и тот же блок (компоненты/шаги/метрики) не описывать дважды.

**6. Исторический раздел** — проверить: «читатель пришёл решить задачу X — этот абзац помогает?». Если нет — убрать.

**7. Глагольные формы в шагах** — одна форма на весь список (инфинитив или повелительное, не вперемешку).

**8. Пояснение к списку** — после него отдельным абзацем, не внутри пунктов.

## СТОП-КОНТРОЛЬ — написать явно перед публикацией

```
Проверка: тире=X, sidenote=X, block-comment=не_тронуты,
is-lead=есть, FAQ=details/summary+JSON-LD, tooltips=X, красных_флагов=0,
H2_дублей=0, суррогатных_меток=0, дублей_расшифровки=0
```
Без отчёта статья считается непроверенной.
