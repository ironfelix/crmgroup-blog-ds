Пример — статья про CRM-маркетолога

Чем занимается CRM-маркетолог?
CRM-маркетолог выстраивает систему коммуникаций с клиентами через email, SMS, мессенджеры и push. Его задача — удерживать клиентов и повышать их ценность для бизнеса. В отличие от email-маркетолога, работает сразу со всеми каналами и отвечает за стратегию.
Когда компании нужен CRM-маркетолог?
CRM-маркетолог нужен, когда: база контактов превышает 10 000 человек, есть 3+ активных канала коммуникации, компания теряет клиентов после первой покупки. До этого порога достаточно email-маркетолога или агентства.
В чём разница между CRM-маркетологом и email-маркетологом?
Email-маркетолог работает только с рассылками. CRM-маркетолог охватывает все каналы (email, SMS, push, мессенджеры), строит сценарии на основе поведения клиентов и отвечает за LTV и удержание, а не только за метрики писем.
Сколько стоит нанять CRM-маркетолога?
Инхаус CRM-маркетолог в Москве стоит 120–180 тыс. ₽/мес (зарплата + налоги + инструменты). Агентство — от 80 тыс. ₽/мес, при этом вы получаете команду: стратег, копирайтер, верстальщик, аналитик. Гибридная модель — оптимальна для большинства.
Как измерить эффективность CRM-маркетинга?
Ключевые метрики: LTV (пожизненная ценность клиента), RR (коэффициент удержания), CR (отток), ROMI (возврат на инвестиции в маркетинг). Смотреть в динамике: квартал к кварталу, а не месяц к месяцу.

🔴 Критично
line-height 1.7 — слишком свободно
Межстрочный интервал 1.7 при font-size 16px создаёт ощущение «воздуха без смысла». Строки не считываются как единый блок, глаз теряет позицию. Оптимум для кириллицы — 1.55–1.6.
/* Было */ line-height: 1.7;
/* Стало */ line-height: 1.58;
/* Исключение: */ .block-comment p { line-height: 1.65; }
🟡 Важно
Нет визуальной иерархии между H2 и H3
H2 = 24px bold, H3 = 18px bold — разница в 6px. На мониторе после 3–4 секций они сливаются. H3 нужно визуально отличать: цветом, весом или отступами.
h3.wp-block-heading {
font-size: 17px;
font-weight: 600; /* не 700 */
color: #3d3d3d; /* чуть светлее h2 */
padding-left: 12px;
border-left: 2px solid #e8e8e8;
}
🟡 Важно
Ширина строки слишком большая
Контент тянется до ~780px. При 16px это ~90–95 символов в строке. Комфортная читаемость кириллики — 65–75 символов (580–640px). Читать длинные строки утомительно.
/* Было */ max-width: 780px;
/* Стало */ max-width: 660px;
/* Сайдбар при этом расширить с 252 до 280px */
🟡 Важно
Фон страницы белый — нет разделения зон
Статья на белом фоне без боковых отступов — нет ощущения «читалки». Лёгкий серый фон (#f8f9fa) и белая карточка контента создают фокус и снижают нагрузку на глаза.
body { background: #f5f5f5; }
.article-inner {
background: #fff;
border-radius: 12px;
padding: 48px 52px;
box-shadow: 0 1px 8px rgba(0,0,0,.06);
}
🟡 Важно
Отступы между блоками хаотичные
fsb-flexible-spacer используется 156 раз — каждый редактор сам расставляет отступы. Нет единой системы: где-то 40px, где-то 24px. Нужна шкала 8px (8, 16, 24, 32, 48).
/* Система отступов 8px */
--space-1: 8px; /* внутри блока */
--space-2: 16px; /* между элементами */
--space-3: 24px; /* между блоками */
--space-4: 40px; /* между секциями */
--space-5: 64px; /* между крупными секциями */

Для статей 2000+ слов оптимально 17–18px. Меньше напряжения, особенно на ретине. 16px уместен для UI, но в теле длинного текста глаза устают быстрее.

🟢 Улучшение
Таблицы без hover и без responsive
На мобиле таблицы не адаптируются — текст обрезается. Нет hover на строках. Добавить горизонтальный скролл и выделение строки при наведении.
figure.wp-block-table { overflow-x: auto; }
tbody tr:hover td {
background: rgba(220,19,39,.04);
}

Нет «Читайте также» между секциями
Внутренняя перелинковка только в конце статьи. Вставка 1–2 ссылок внутри текста и блока «Читайте также» в середине снизит отказы и улучшит поведенческие факторы для Яндекса.

TOC без прогресс-индикатора
navigation-page показывает разделы, но не подсвечивает текущий. Sticky TOC с active-состоянием держит читателя в контексте и снижает показатель отказов.

Нет og:image в статьях
При шаринге в Telegram/VK нет превью-картинки — пост выглядит голым текстом. Добавить og:image 1200×630px для каждой статьи через Yoast или кастомное поле.
<meta property="og:image"
