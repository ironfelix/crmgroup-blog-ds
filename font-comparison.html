4. Длинный текст (имитация статьи, ~660px)

Inter (текущий)

Что такое Customer Value Management

CVM (Customer Value Management) — это стратегический подход, при котором компания выстраивает маркетинг вокруг ценности каждого клиента. Вместо массовых рассылок и одинаковых скидок для всех, CVM сегментирует базу по прогнозируемому LTV и подбирает коммуникации индивидуально.

По данным McKinsey, компании, внедрившие CVM-подход, увеличивают доход от существующей базы на 15–25% за первый год. При этом стоимость удержания клиента в 5–7 раз ниже стоимости привлечения нового.

Ключевые метрики CVM:

  • LTV (Lifetime Value) — совокупный доход от клиента за весь период
  • Retention Rate — процент клиентов, совершивших повторную покупку
  • Churn Rate — процент оттока за период
  • CAC (Customer Acquisition Cost) — стоимость привлечения

Формула окупаемости проста: если LTV > CAC * 3, бизнес-модель устойчива. Если LTV < CAC, компания теряет деньги на каждом клиенте.

Вердикт типографа

TTSmalls as-is (тема) — нечитабелен из-за line-height: normal (~1.2) при 18px. Строки слипаются. Кириллические буквы с высокими выносными элементами (б, д, ф) цепляют нижнюю строку.

TTSmalls optimized (ВЫБРАН для хаба) — читабельность на уровне Inter после настройки:

  • Size: body 17px, desc 17px (TTSmalls крупнее Inter при одинаковом px)
  • Line-height: normal → 1.65 (больше Inter 1.58 из-за конструкции шрифта)
  • Letter-spacing: 0 → 0.008em (TTSmalls плотнее Inter)
  • Font smoothing: auto (subpixel), НЕ antialiased — TTSmalls-400 слишком тонкий с antialiased
  • Headings: TTRuns 700, letter-spacing -0.03em, line-height 1.15
  • Card headings: line-height 1.35, letter-spacing -0.01em
  • Meta text (author, categories): font-weight 500 (Medium) для плотности
  • text-wrap: balance на заголовках, pretty на описаниях

Inter (ВЫБРАН для статей) — DS v6: 17px, line-height 1.58, Inter.

Решение (2026-02-22): Хаб /blog/ → TTSmalls/TTRuns optimized (фирменный шрифт темы). Статьи .single-blog → Inter (DS v6). Визуальный "скачок" при переходе хаб→статья незаметен, т.к. хаб = карточки/навигация, статья = длинное чтение — разный контекст.

Дополнительные решения (2026-02-22)

  • Line-clamp 3 на .article-desc — описания обрезаются, карточки выравниваются по высоте
  • Equal-height cards — flex column + margin-top:auto на .button-read
  • Чекбоксы 18x18px вместо 12x12 — больше touch target, border-radius 3px
  • Валидация без дёрганья — .wpcf7-not-valid-tip position:absolute
  • Висячие предлоги — &nbsp; в WP excerpts (CSS text-wrap не решает для русских предлогов)
  • Мобилка без картинки — слайдер: вертикальный центр, gradient background, font-size 36px
