4. Длинный текст (имитация статьи, ~660px)

Inter (текущий)

Что такое Customer Value Management

CVM (Customer Value Management) — это стратегический подход, при котором компания выстраивает маркетинг вокруг ценности каждого клиента. Вместо массовых рассылок и одинаковых скидок для всех, CVM сегментирует базу по прогнозируемому LTV и подбирает коммуникации индивидуально.

По данным McKinsey, компании, внедрившие CVM-подход, увеличивают доход от существующей базы на 15–25% за первый год. При этом стоимость удержания клиента в 5–7 раз ниже стоимости привлечения нового.

Ключевые метрики CVM:

  • LTV (Lifetime Value) — совокупный доход от клиента за весь период
  • Retention Rate — процент клиентов, совершивших повторную покупку
  • Churn Rate — процент оттока за период
  • CAC (Customer Acquisition Cost) — стоимость привлечения

Формула окупаемости проста: если LTV > CAC * 3, бизнес-модель устойчива. Если LTV < CAC, компания теряет деньги на каждом клиенте.

Вердикт типографа

1. Честная оценка: что умеет каждый шрифт

Inter — спроектирован Расмусом Андерссоном специально для экранов. Открытые апертуры (буквы «a», «e», «c», «s» с широким просветом), ровная толщина штрихов, высокий x-height. Это значит: текст читается быстро, глаз не цепляется, даже при 14-15px всё разборчиво. Для лонгридов на экране — это объективно один из лучших шрифтов в мире. Бесплатный (OFL).

TTSmalls — геометрический гротеск от TypeType, спроектирован для брендинга и навигации. Плотная посадка букв, тонкий штрих в Regular (400), закрытые апертуры. Хорош для заголовков карточек, меню, кнопок — везде, где текст короткий и нужен фирменный характер. Для лонгридов — проигрывает Inter по всем метрикам читаемости. Коммерческая лицензия (уже куплена для темы).

TTRuns — акцидентный (display) шрифт от TypeType. Сжатые пропорции, сильный характер. Отлично работает в заголовках H1-H2, лого-блоках, hero-секциях. Не предназначен для мелкого текста. Коммерческая лицензия.

Главный вывод: Заставить TTSmalls читаться как Inter в лонгридах — это как заставить спорткар ехать комфортно по грунтовке. Можно подкачать подвеску (line-height, word-spacing, hyphens), и будет терпимо. Но бизнес-класс (Inter) всё равно будет комфортнее — он для этого сделан.

2. Рекомендация: двушрифтовая система TT + Inter

Два шрифта — это не баг, а стандартная практика. Medium, Stripe, Linear, Notion — все используют акцидентный шрифт для бренда + нейтральный для чтения. Ключ в том, чтобы переход между ними был осознанным, а не случайным.

3. Как связать TT и Inter, чтобы не было «скачка»

Переход шрифта происходит при клике на карточку: хаб (TT) → статья (Inter). Три приёма, чтобы это ощущалось как единый дизайн, а не два разных сайта:

A. Общая цветовая и пространственная система

  • Один и тот же акцент #c01020, одна палитра серых, одни отступы (8px-grid)
  • Одинаковая ширина контентной колонки (660px) и структура хедера
  • Читатель не замечает смену шрифта, если всё остальное стабильно

B. Shared metrics (совпадающие метрики)

  • Body size: TTSmalls hub 16px ≈ Inter article 17px (TT визуально крупнее при одинаковом px из-за x-height)
  • Line-height: TT 1.62 ≈ Inter 1.58 (визуально одинаковый «воздух», т.к. у TT выше строчные)
  • Card titles: TTRuns 18px ↔ Inter H3 17px (визуально совпадают по жирности и ритму)
  • Принцип: подбираем числа не одинаковые, а визуально эквивалентные

C. Transition point = смена контекста

  • Хаб — это навигация (сканирование, выбор). TT подходит: короткие тексты, фирменный стиль
  • Статья — это погружение (чтение 10+ минут). Inter подходит: комфорт, ритм, ровное дыхание
  • Переключение шрифта совпадает с переключением режима поведения пользователя — поэтому оно незаметно
  • Аналогия: вывеска магазина (бренд-шрифт) → книга на полке (читабельный шрифт). Никто не ожидает, что они совпадут

4. А если всё-таки хочется TT для статей?

Единственный сценарий, когда TTSmalls в лонгридах оправдан: бренд-гайдлайн запрещает другие шрифты (например, для франшизы или строгого корпстиля). В этом случае — вариант EXPERT:

Приём Значение Что даёт Вклад в читаемость
font-smoothing: auto subpixel Утолщает тонкий штрих TT-400 +++ (критично)
word-spacing 0.05em Чёткие границы слов +++ (главный приём)
line-height 1.68 Воздух между строк ++ (заметно)
letter-spacing 0.01em Разрежает плотную посадку ++ (заметно)
hyphens: auto авто-переносы Ровный правый край, нет «дыр» + (на длинных абзацах)
text-rendering optimizeLegibility Кернинг-пары шрифта + (тонкая разница)
фон #faf9f6 тёплая бумага Снижает усталость от контраста + (комфорт, не читаемость)
text-wrap: pretty умные переносы Меньше «сирот» в конце строк + (эстетика)

Обновление 2026-02-23: EXPERT-свойства были протестированы на продакшне (Inter в статьях) и убраны. Причины:
• text-wrap: pretty — перераспределяет строки, вызывает «пляску» концов строк
• text-rendering: optimizeLegibility + font-feature-settings — меняют кернинг → другие переносы
• hanging-punctuation — сдвигает пунктуацию за поля → визуально неровно
• word-spacing: 0.05em — помогает TTSmalls, но ухудшает Inter (у него и так хороший word-spacing)
• hyphens: auto — русские авто-переносы некрасивые: «кли-», «цен-», «о-»

Вывод: Inter не нуждается в дополнительном тюнинге — он уже оптимально рендерится из коробки. Эти приёмы полезны для TTSmalls (у него слабее дефолтный кернинг), но не для Inter.

С EXPERT-настройками TTSmalls набирает ~85-90% читаемости Inter. Для 80% читателей разницу будет сложно артикулировать — «вроде нормально читается». Но последние 10-15% (открытые апертуры, равномерность штриха) — это ДНК шрифта, которую CSS не изменит.

5. Итоговая рекомендация

Вариант A: TT + Inter (рекомендую)

  • Hub, навигация, хедер, карточки → TTSmalls/TTRuns opt
  • Статьи, лонгриды, FAQ, CTA → Inter
  • Визуальный «мост»: общие цвета, spacing, ширина колонки
  • Переход хаб→статья = смена контекста (навигация→чтение)

Плюсы: максимальный комфорт чтения + фирменный стиль на хабе.
Минусы: два шрифта = два кэша (Inter из Google Fonts, TT из темы). Плюс ~40KB.

Вариант B: Только TT (если бренд требует)

  • Всё → TTSmalls EXPERT + TTRuns
  • Обязательно: subpixel, word-spacing, lh 1.68, hyphens
  • Опционально: тёплый фон #faf9f6 для статей
  • Заголовки статей: TTRuns 700 (вместо Inter 700)

Плюсы: единый бренд-шрифт на всём сайте. Один кэш.
Минусы: ~10-15% потери читаемости в лонгридах. Чувствуется на статьях 2000+ слов.

Моя позиция: вариант A. Читатель проводит 10 секунд на хабе и 10 минут в статье. Оптимизировать нужно то, где человек проводит больше времени. TT на хабе даёт узнаваемость бренда, Inter в статье даёт комфорт — каждый шрифт делает то, для чего создан.

6. CSS-рецепт

Вариант A — двушрифтовая система:

/* Hub: фирменный TT */
#blog { font-family: "TTSmalls", sans-serif; }
#blog h1, #blog h2, #blog h3,
#blog .card-title { font-family: "TTRuns", sans-serif; }
#blog p, #blog .desc {
  font-size: 16px; line-height: 1.62;
  letter-spacing: 0.005em;
  -webkit-font-smoothing: auto; /* subpixel */
}

/* Статья: читабельный Inter */
.single-blog .section-content {
  font-family: Inter, -apple-system, sans-serif;
  -webkit-font-smoothing: antialiased;
}
.single-blog .section-content p {
  font-size: 17px; line-height: 1.58;
}

/* Мост: общие токены */
:root {
  --accent: #c01020;
  --text: #1a1a1a;
  --reading-width: 660px;
}

Вариант B — только TT с EXPERT:

/* Всё: TTSmalls EXPERT */
body {
  font-family: "TTSmalls", sans-serif;
  -webkit-font-smoothing: auto;
  text-rendering: optimizeLegibility;
  font-feature-settings: "kern" 1, "liga" 1, "calt" 1;
}
h1, h2, h3 {
  font-family: "TTRuns", sans-serif;
  text-wrap: balance;
}

/* Лонгрид: максимальная оптимизация */
.single-blog .section-content {
  background: #faf9f6;
}
.single-blog .section-content p {
  font-size: 17px;
  line-height: 1.68;
  letter-spacing: 0.01em;
  word-spacing: 0.05em;
  hyphens: auto;
  text-wrap: pretty;
  color: #222;
  hanging-punctuation: first last;
}
